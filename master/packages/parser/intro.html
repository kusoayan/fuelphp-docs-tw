Parser 套件

Parser 套件擴展核心視圖類別，能夠使用任何你想要的樣板解析器。

簡介

Parser 套件不改變任何你使用視圖的方法，但它將能讓你使用任何你想要的樣板解析器取代 PHP。這套件有著許多解析器的驅動，像 Mustache、Markdown、Twig 和 Smarty。

使用哪個解析器取決於你的視圖副檔名。通常視圖副檔名為 .php 且會使用 PHP 解析樣板, 這些 .php 副檔名也可以被省略。所有的驅動都有自己特定的副檔名，例如 foo.mustache 將使用 Mustache 驅動，而 foo.tpl 將使用 Dwoo 驅動（預期副檔名可在配置中被改變）。

安裝

Parser 套件包含在 Fuel 下載中，要使用你必須先添加啟用至你的配置：

'packages' => array(
	'parser',
),

雖然很多驅動被包含在內，大多數常式庫沒有。只有 Mustache 與 Markdown 被包含在套件的 vendor 目錄，且開箱即用。 其他解析器你必須從他們的網站下載，如果你想要在不改變他們被預期在 app/vendor/{Lib_name} （其中 Lib_name 的第一個字符是大寫的）的配置下運行。

配置

Parser 能在 config/parser.php 檔案中被配置，且使用以下的鍵：

變數 類型 預設 描述
extensions 陣列 （包含擴展與解析的視圖類別的陣列） 定義哪些擴展是有效的以及他們的視圖驅動是什麼。

執行階段配置

所有驅動都有一個 parser() 方法能讓你訪問當前的 Parser 物件。這是一個慣例而非必要條件，且不同的第三方驅動程序可能會有所不同。

// 清除特定 Smarty 樣板的快取
$view = View::forge('example.smarty');
$view->parser()->clearCache('example.smarty');

// 靜態用法的範例
View_Smarty::parser()->clearCache('example.smarty');
