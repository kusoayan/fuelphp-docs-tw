命令列安裝

目前只能在 *nix 系統（Linux、OS X、Unix、等等）運行。

我們的快速安裝程序是一個精簡的 Oil 套件介面，能讓你一行命令建立一個新的專案，你也將不再需要在你的 Oil 命令使用 'php'。

要安裝快速安裝程序，只要打開一個 shell 並執行以下命令：

$ curl get.fuelphp.com/oil | sh

這將詢問你的密碼，因為它安裝腳本到 /usr/bin。

現在你可以在你的專案裡使用 'oil' 取代 'php oil'。

要建立一個新的專案，只要運行：

$ oil create <project_name>

這將在你給定名稱的目錄裡建立一個文件夾，接著複製容器與所有子模組到該目錄。

注：這也將運行 $ oil refine install，使必要的目錄為可寫。

手動安裝簡介

這將在你的網頁伺服器建立 Fuel 框架的預設安裝。

  1. 下載 Fuel 框架
  2. 解壓縮／抽取下載的檔案
  3. 移動檔案到你的伺服器
    • 注意在程式碼裡的 public 目錄，等於你的網頁伺服器的公開文件目錄，也就是，public_html、public、htdocs等。移動其內容到這些目錄，或一個你要使用 Fuel 的網頁根目錄下的子目錄。
    • 促進安全性原因，放置 Fuel 目錄於公開文件目錄之外。
    • 編輯 index.php 裡的路徑指向你的應用程式、核心＆套件目錄。
    /
  fuel/
    app/
    core/
    packages/
  public/
    .htaccess
    assets/
    index.php
  oil
  4. 手動修改文件夾的權限為可寫，或運行： 
    $ php oil refine install
	Made writable: APPPATH/cache
	Made writable: APPPATH/logs
	Made writable: APPPATH/tmp

配置

主要的配置在 app/config/config.php，根據你的喜好編輯它。

安裝在文件根目錄

As explained in point 3, for security reasons it is strongly advised NOT to install Fuel inside your webserver's document root.

However, there are cases where you would like to do that, for example for a (local) development environment where Apache's dynamic mass virtual hosting module is used to quickly setup new development environments without the need to restart the webserver.

In that case, you need an additional .htaccess file that you need to place in your document root, which will redirect requests to the site root to your public folder, and also modifies the rewrites to include the public folder:


<IfModule mod_rewrite.c>
	RewriteEngine on

	RewriteBase /public

	RewriteRule ^(/)?$ index.php/$1 [L]

	RewriteCond %{REQUEST_FILENAME} !-f
	RewriteCond %{REQUEST_FILENAME} !-d

	RewriteRule ^(.*)$ index.php/$1 [L]
</IfModule>

設定環境

預設情況下，環境設定為開發模式。Fuel 使用環境去定義使用什麼資料庫設定，但你可以在其他東西使用它。

要設定環境，將以下行拖放進你的 .htaccess 檔案。 

SetEnv FUEL_ENV production

詳細可用選項在 Fuel 類別裡的 Class constants。
